深度学习炼丹遇到的问题总结
Created2021-03-24|Updated2021-05-01
|Post View:

1.RuntimeError: Expected object of backend CUDA but got backend CPU for argument #3 'index' ： 这种情况是因为某些参数/数据没有放到gpu上，仔细检查一下

2.RuntimeError: arguments are located on different GPUs at /pytorch/aten/src/THC/generic/THCTensorIndex.cu:521: 这种情况是因为同一个函数的参数却放到了不同的gpu上，此处需注意data.cuda()和data.to(device)是不一样的——cuda()会默认gpu 0，而to()会指定device

3.AttributeError: 'list' object has no attribute 'cuda':一看就知道忘了把list转成tensor，根据list内元素不同,可以使用torch.tensor(),torch.stack()等等

3.RuntimeError: CUDA out of memory. Tried to allocate 48.00 MiB (GPU 6; 11.93 GiB total capacity; 11.11 GiB already allocated; 4.06 MiB free; 311.56 MiB cached):非常经典的报错，一般是由于batch数据太大,超出了GPU的显存，需要不断尝试减小batch_size或者修改模型的loss计算方式，比如list-wise变为pair-wise或者point-wise

4.IndexError: Dimension out of range (expected to be in range of [-1, 0], but got 1):这种情况是维度出现了错误，要处理的tensor一维(dim=0)，并没有dim=1，检查一下代码，判断是否要加unsqueeze(dim=1)

5.RuntimeError: invalid argument 6: wrong matrix size at /pytorch/aten/src/THC/generic/THCTensorMathBlas.cu:494:矩阵乘法操作中两个矩阵维度不对应，仔细检查一下二者的维度以及对应torch.matmul/bmm/mul的维度要求

6.RuntimeError: merge_sort: failed on 2nd step: invalid argument: 不知道为什么报这个错。。。解决是多运行几次python xx.py即可。。
